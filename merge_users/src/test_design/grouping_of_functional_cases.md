# Группировка тест-кейсов

## Подготовка данных

### users.txt
Позитивные:
- файл существует
- один логин на строку
- несколько пользователей

Негативные:
- файл отсутствует
- пустой файл


### users.json
Позитивные:
- JSON содержит login:name
- несколько пользователей

Негативные:
- файл отсутствует
- пустой JSON
- некорректный JSON


### users.csv
Позитивные:
- CSV содержит header login,email
- корректные email

Негативные:
- файл отсутствует
- пустой CSV
- отсутствует разделитель

# Запуск скрипта

- exit code 1

# Проверка результата full_users.csv
Проверить:

- наличие header login,name,email
- наличие пользователей
- отсутствие пользователей без данных
- отсутствие дублей
