# Шаги тест кейса

1) Подготовка данных (3 файла - users.txt, users.csv, users.json)
- создание файла users.txt
- установка конкретных значений пользователей (user1, user2, user3)
-
- создание файла users.csv
- установка header (login,email)
- установка конкретных данных (user, email)
- - ВАЖНО: user должен существовать в users.txt
- создание файла users.json
- установка конкретных данных (user, name)
- - ВАЖНО: user должен существовать в users.txt

2) Запуск скрипта
- передача параметров запуска
-  - usersTXTFilePath
-  - usersCSVFilePath
-  - usersJSONFilePath
- успешный запуск скрипта (exit code 0)
- неуспешный запуск скрипта (exit code 1)

3) Проверка результата в файле full_users.csv
- проверка содержимого файла full_users.csv (построчно)
- проверка сообщений ошибок

## BDD шаблон теста
Тест кейс "Название"

#### given

- подготовка входных файлов
- установка конкретных данных

#### when

- запуск скрипта
- проверка exit code

#### then

- проверка full_users.csv
- проверка ошибок
