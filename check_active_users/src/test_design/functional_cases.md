# Функциональные тест-кейсы check_active_users.sh
1) Один активный пользователь
2) Несколько активных пользователей
3) Заблокированный пользователь исключается
4) Пользователь старше 30 дней исключается
5) Пользователь ровно 30 дней назад включается
6) Пользователь отсутствует в logins.csv
7) Пустой users.txt
8) Пустой banned.json
9) Некорректная дата в logins.csv
10) Пустой last_login
11) Отсутствует users.txt
12) Отсутствует logins.csv
13) Отсутствует banned.json
14) Невалидный JSON в banned.json
15) Проверка формата active_users.csv

## Тест кейс "Один активный пользователь"
given
- создание файла users.txt
- установка пользователя alice

- создание файла logins.csv
- установка header (login,last_login)
- установка данных (alice, 2025-06-20)

- создание файла banned.json
- установка пустого списка блокировки

when
- запуск скрипта с параметрами:
    - usersFilePath
    - loginsFilePath
    - bannedFilePath
- успешный запуск скрипта (exit code 0)

then
- проверка создания файла active_users.csv
- проверка содержимого active_users.csv:
```csv
login,last_login
alice,2025-06-20
```
