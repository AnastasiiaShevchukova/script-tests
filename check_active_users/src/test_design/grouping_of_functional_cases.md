# Группировка тест-кейсов

## Проверка передачи и обработки входных файлов

Пример входных данных:
- users.txt

```txt
alice
bob
carol
```
logins.csv
```csv
login,last_login
alice,2025-06-20
bob,2025-05-01
carol,2025-01-01
```
banned.json
```json
[
"bob"
]
```
---

### Проверка наличия входных файлов
### users.txt

- Позитивный сценарий: файл существует, файл содержит список логинов
- Негативный сценарий: файл отсутствует, передан неверный путь


### logins.csv

- Позитивный сценарий: CSV файл существует, файл содержит данные пользователей и даты входа
- Негативный сценарий: файл отсутствует, файл пустой


### banned.json

- Позитивный сценарий: JSON файл существует, содержит массив заблокированных пользователей
- Негативный сценарий: файл отсутствует, JSON имеет неверный формат


---


### Проверка обработки формата users.txt
### Формат данных пользователя

- Позитивный сценарий: один логин на строку
- Позитивный сценарий: несколько пользователей в файле

- Негативный сценарий: пустые строки
- Негативный сценарий: некорректный логин


## Количество пользователей

- Позитивный сценарий: один пользователь
- Позитивный сценарий: несколько пользователей

- Негативный сценарий: пустой файл


---


### Проверка обработки формата logins.csv


### Разделитель CSV

- Позитивный сценарий: используется разделитель `,`

- Негативный сценарий: используется другой разделитель `;`
- Негативный сценарий: отсутствует разделитель


### Первая колонка login

- Позитивный сценарий: логин существует в users.txt

- Негативный сценарий: логин отсутствует в users.txt


### Вторая колонка last_login

- Позитивный сценарий: дата корректного формата YYYY-MM-DD

- Негативный сценарий: дата некорректного формата
- Негативный сценарий: пустое значение даты


---

### Формат JSON

- Позитивный сценарий: JSON содержит массив логинов

Пример:

```json
[
"bob",

]
```
