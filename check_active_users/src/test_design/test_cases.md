# Шаги тест кейса

1) Подготовка данных (3 файла - users.txt, logins.csv, banned.json)
- создание файла users.txt 
- установка конкретных значений пользователей (user1, user2, user3)
- 
- создание файла logins.csv
- установка header (login,last_login)
- установка конкретных данных (user, last_login)
- - ВАЖНО: user должен существовать в users.txt
- создание файла banned.json
- установка конкретных пользователей в список блокировки

2) Запуск скрипта
- передача параметров запуска
-  - usersFilePath
-  - loginsFilePath
-  - bannedFilePath
- успешный запуск скрипта (exit code 0)
- неуспешный запуск скрипта (exit code 1)

3) Проверка результата в файле active_users.csv
- проверка содержимого файла active_users.csv (построчно)
- проверка сообщений ошибок

## BDD шаблон теста
Тест кейс "Название"

#### given

- подготовка входных файлов
- установка конкретных данных

#### when

- запуск скрипта
- проверка exit code

#### then

- проверка active_users.csv
- проверка ошибок
